# Architecture Overview - Blog Writing Agent

The Blog Writing Agent (BWA) is built using **LangGraph**, providing a cyclic, stateful multi-agent system for autonomous blog post generation, orchestrated via a **FastAPI** backend and visualized on a **React (Vite)** frontend.

## Graph Structure

The core application logic (`backend/bwa_backend.py`) follows a specialized workflow:

1.  **Router**: Analyzes the topic and decides if web research is required. It sets the `mode` (closed_book, hybrid, open_book).
2.  **Research (Optional)**: Uses **Tavily Search** to gather up-to-date evidence for the topic.
3.  **Orchestrator**: Defines a detailed `Plan` for the blog post, including specific tasks and section target words.
4.  **Worker (Fan-out)**: Launches parallel agents to write individual sections of the blog based on the plan and collected evidence.
5.  **Reducer (Sub-graph)**: Merges the sections and performs a post-processing flow for images:
    *   **Merge Content**: Combines sections into a single markdown document.
    *   **Decide Images**: Determines where technical diagrams or images would improve the post.
    *   **Generate/Place Images**: Uses **Gemini 2.5 Flash Image** to generate images and inserts them into the markdown.

## State Management

The graph maintains a global `State` TypedDict:
- `topic`: The user-defined subject.
- `plan`: The structured outline generated by the Orchestrator.
- `evidence`: List of `EvidenceItem` objects gathered from research.
- `sections`: A list of tuples `(task_id, section_md)` that accumulates as Workers finish.
- `final`: The complete, rendered markdown string.

## Client-Server Interactions
The graph execution state is streamed in real-time to the frontend.
- **Backend**: `backend/api.py` exposes exactly one endpoint `/generate` that yields Server-Sent Events (SSE) detailing the payload of each invoked node.
- **Frontend**: The React application parses the stream to provide live UX feedback on the currently running agent.

## Dependencies

- **LangGraph & LangChain**: Orchestration engine.
- **FastAPI & sse-starlette**: Backend web server and streaming provider.
- **React (Vite)**: Glassmorphic user interface.
- **Google Generative AI**: LLM (`gemini-1.5-flash`) and Image generation.
- **Tavily**: Web search capabilities.
